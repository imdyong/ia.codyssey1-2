# \[Brand Identity] AeroNode: 당신의 시간을 되찾는 기술

## 1\. 🚀 브랜드 개요 및 설정 근거

\*\*AeroNode(에어로노드)\*\*는 AI 네트워크 최적화 기술로 출퇴근 정체를 0%로 줄이는 **클린 테크(Clean-tech) 모빌리티 플랫폼**입니다.

### 1-1. 타겟 오디언스 및 선정 근거

|항목|내용|선정 근거|
|-|-|-|
|연령대|30대 직장인|통계청 「통근 실태 조사(2024)」 기준 수도권 30대 평균 통근 시간 왕복 116분으로 전 연령대 중 최장. 시간 절약 니즈가 가장 큰 집단|
|특징|스마트 라이프스타일 추구 도심 거주자|모빌리티 앱 사용률(30대 78%, 오픈서베이 2024)이 가장 높아 신규 플랫폼 수용성이 높음|

### 1-2. 핵심 가치 (USP)

**"AI 기반 실시간 네트워크 최적화 → 출퇴근 정체 0% 구현"**

* 단순 길 찾기가 아닌, 도시 전체 교통 흐름을 AI가 분석해 정체 구간을 **사전에 회피**하는 경로 제공
* 근거: 기존 내비게이션은 '현재 정체'만 반영 → AeroNode는 '예측 정체'까지 반영한다는 차별점

### 1-3. 톤앤매너 및 선택 근거

|요소|내용|선택 근거|
|-|-|-|
|톤|미래 지향적, 클린 테크|경쟁사(카카오T·티맵)의 캐주얼 톤과 차별화, 기술 신뢰감 강조|
|컬러|블루(#1A73E8) / 화이트(#FFFFFF)|블루=신뢰·기술(핀테크/테크 업계 표준), 화이트=클린 테크 이미지. 레퍼런스: Tesla, Waymo 브랜드 팔레트|
|스타일|미니멀리즘|"복잡함을 덜어낸다"는 브랜드 메시지와 시각 언어의 일치|

## 7\. 🎛️ 기술 선택 근거

### 7-1. T2I vs I2V 차이 및 본 프로젝트 선택 근거

|구분|T2I (Text→Image)|I2V (Image→Video)|T2V (Text→Video)|
|-|-|-|-|
|특징|텍스트로 정지 이미지 생성|이미지를 기반으로 모션 생성|텍스트만으로 영상 생성|
|장점|구도·색감 정밀 제어 가능|키프레임 기반이라 스타일 일관성 유지|빠른 제작|
|단점|움직임 없음|모션 범위 제한적|씬 간 스타일·색감 통제 어려움|

**선택: T2I → I2V 조합.** 이유: 4씬 모두 블루/화이트 미니멀 톤을 유지해야 하는데, T2V는 씬마다 스타일이 달라지는 문제가 있음. T2I로 키프레임을 먼저 확정·검수한 뒤 I2V로 모션화하면 브랜드 일관성을 보장할 수 있음.

### 7-2. 도구 선택 기준 및 대체 도구 비교 (품질·속도·비용)

|도구|품질|속도|비용|선택 여부|
|-|-|-|-|-|
|Midjourney v6 (T2I)|상 (톤 일관성 우수)|중 (약 1분/장)|$10/월|✅ 채택|
|DALL·E 3 (T2I)|중 (텍스트 반영 우수, 실사감 낮음)|상|ChatGPT 구독 포함|대체 1순위|
|Leonardo.ai (T2I)|중|상|무료 크레딧 150/일|대체 2순위|
|Runway Gen-3 (I2V)|상 (자연스러운 카메라 모션)|중 (약 2분/클립)|무료 125크레딧|✅ 채택|
|Pika 1.5 (I2V)|중|상|무료 크레딧 제공|대체 1순위|
|Kling AI (I2V)|상 (단, 대기열 김)|하|무료 일일 크레딧|대체 2순위|

## 8\. 🎨 스타일 제어 및 검수

### 8-1. 레퍼런스·파라미터

* **비주얼 레퍼런스**: Tesla 브랜드 필름, Apple 제품 영상(미니멀·화이트 배경)
* **Midjourney 파라미터**: `--ar 16:9 --style raw --stylize 250` (전 씬 동일 적용해 스타일 통일)
* **Runway 설정**: 카메라 모션 low, 모션 강도 5/10 (과한 왜곡 방지)

### 8-2. 검수(후처리) 체크리스트

* \[ ] 블루 #1A73E8 계열 컬러가 씬 내 존재하는가
* \[ ] 인물·차량 형태 왜곡(손가락, 바퀴 등) 없는가
* \[ ] 씬 간 색 온도 차이가 크지 않은가
* \[ ] 텍스트/로고가 깨지지 않았는가 (S4)
* \[ ] 16:9, 1920x1080 규격 준수 여부

### 8-3. 툴 통합 시 해상도·색감 보정 절차

1. Runway 출력(1280x768) → CapCut에서 **1920x1080 업스케일**
2. CapCut 색 보정: S1은 채도 -10(회색 톤), S3·S4는 색온도 +5·밝기 +10으로 통일감 부여
3. 씬 전환부에 0.3초 크로스 디졸브 적용해 색감 차이 완화
4. 오디오: 내레이션 0dB 기준, BGM -12dB로 믹싱

## 9\. ⚠️ 리스크 대응 전략

### 9-1. 생성 시도 제한 및 대체 계획

* 씬당 T2I 최대 **3회**, I2V 최대 **2회** 시도. 초과 시 대체 도구(7-2 표)로 전환

### 9-2. 크레딧 부족 시 대책 (우선순위 순)

1. **씬 단축**: S2+S3 통합 → 3씬 구성으로 축소 (I2V 크레딧 25% 절감)
2. **도구 변경**: Runway → 무료 크레딧이 남은 Pika/Kling으로 전환
3. **편집 전략**: I2V 실패 씬은 정지 이미지에 CapCut 줌/팬 효과(Ken Burns)로 대체

### 9-3. 타임라인 단축 시 유지/삭제 기준

* **반드시 유지**: S4 브랜딩 씬(슬로건), S2 솔루션 씬(USP) — 핵심 메시지 "복잡한 도심을 건너, 당신의 시간을 되찾다"의 전달에 필수
* **삭제 가능**: S1 도입부(문제 제기는 내레이션만으로 대체 가능)
* **재구성 전략**: 10초 초과 시 S1 삭제 → "AI가 흐름을 읽고(S2) → 도시를 건너(S3) → 시간을 되찾다(S4)" 3단 구조로 메시지 재편
